# OpenSpec Changes

Create one folder per non-trivial task or issue:

`openspec/changes/<change>/`

Recommended naming:

- `issue-42-short-kebab-title` for issue-driven work
- `maintenance-short-kebab-title` for internal work without an issue

Each change folder should usually contain:

- `proposal.md`
- `design.md`
- `tasks.md`

Add `specs/<domain>/spec.md` only when the change introduces or modifies normative requirements.
