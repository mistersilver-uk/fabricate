# Design: Manager V2 Follow-ups

## Use Of This Directory

Use this directory as a lightweight parent reference only. New implementation work should be split into focused OpenSpec changes instead of growing this parent backlog.

## Durable References

- `design-system.md` remains the Manager V2 token, surface, and component reference used by runtime CSS and future UI work.
- `references/` remains the visual reference library for targeted Manager V2 and actor-app work.

## Follow-up Defaults

- Keep Manager V2 work route-focused and behavior-preserving.
- Prefer existing Svelte components, stores, callbacks, localization, and `--fab-*` tokens.
- Do not reintroduce legacy Handlebars templates or broad `.fabricate-admin` styling.
- Capture screenshot evidence only for focused follow-up changes that modify user-visible UI.
